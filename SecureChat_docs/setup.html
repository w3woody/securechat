Client Setup

This covers the steps for building and deploying a client, as well as setting up the client device for chatting.

Requirements

The SecureChat iOS client was built with Xcode v7.

Building

  • Open the SecureChat.xcodeproj file in Xcode. SecureChat has no other dependencies.
  • Build the application. You will need an Apple Developer account in order to install the application on your iOS devices; the type of account will determine what limits on the number of devices you have during installation.

Setting up a running client.

All of the steps for setting up the client, as well as steps for adding a new device, for changing the password, and for unregistering a device, are handled through a series of "wizard"-like screens which should help you set up your device. The process should be self-explanatory.

Onboarding Steps

The steps used when starting up a client are outlined below. You must complete these steps prior to using the application.

Select a Passcode

A passcode is a sequence of digits which is used to generate an encryption key for encrypting the contents of the keystore used to store your RSA key. The longer the sequence, the more secure your keystore and the messages on your device. The sequence must be at least 4 digits long.

Generate an RSA Key

All messages sent and received by the device are sent and received using RSA. The longer the RSA key, the more time it takes to generate a key and to encrypt and decrypt messages. In general a 1024 bit key is good enough, but if you are willing to wait longer, you can pick a 2048 bit key or a 4096 bit key. Note that a 2048 bit key can take up to 2 minutes to generate, and a 4096 key can take several minutes.

Select SecureChat Server URL

This will depend on where the SecureChat server was installed. You will need to obtain this URL from the person who installed the server. By default the URL is assumed to be https://, unless specified otherwise.

For example, if the server was installed at http://myserver.com (and that is the URL returned during installation), you may enter:

http://myserver.com

If the server was installed with https://, you can instead enter:

myserver.com

If the wrong URL is entered or the server is not present, you will get an error.

Create an account or log into an existing account

If you have an existing account you may enter the credentials. If you have forgotten your existing account's password, you can request to reset the password, but note this will only work if you have a device currently registered to that account. (Password reset requests are broadcast to devices currently registered to the account, as there are no e-mail addresses stored for each user account.)

Creating an account only requires a username (which you should select in a way which does not identify yourself; i.e., don't use your name), and a password. The password must contain at least 8 letters, including one capital letter, one lowercase letter, one digit and one punctuation character.

Note that all passwords are hashed, and are not stored in clear text.

Done!

Enjoy securely chatting with other users!

Maximizing Security

Here are some steps you can do to make sure the security of your messages are maximized.

Account Security

  • Pick a username which does not identify who you are.
  • Pick a secure password that cannot be easily guessed.
  • Never share your password with anyone. The SecureChat password is hashed, so there is never any need to share your password, and there is no way for others to inspect the server to discover your password.

Device Security

  • Pick a good passcode, one that is hard to guess (like your birthday), but one which is easy for you to remember.
  • Never share your device with anyone.
  • Always pay attention to the number of devices SecureChat thinks you have registered to your account. If that number does not match the devices you think you have, discontinue using your account immediately.

Secure chatting

  • Pay attention to the number of devices registered to the person you are chatting with. If that number suddenly jumps, you may want to contact that person separately and ask them if they've added a new device to their account.
